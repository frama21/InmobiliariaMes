alternativasEstandar.txt

Atributos
-> Palabras completas con mayúsculas en cada palabra no inicial.
-> Ejemplos: nombre // codigoPostal
   
Componentes
-> Palabras completas con sufijos con el nombre del tipo de componente.
-> Ejemplos: okButton // clienteList

Colecciones
-> Nombres plurales. 
-> Ejemplos: clientes // lineasPedido

Variables locales
-> Palabras completas con mayúsculas en cada palabra no inicial. 
-> Exceptuando streams (in/out/inout), bucles (i, j, k...) y excepciones (e / palabras prefijadas con e)
-> Ejemplos: nombre // codigoPostal

NOTA: variables locales declaradas inmediatamente antes de su uso.
NOTA: variables compartidas declaradas al inicio de cada clase.

Parámetros
-> Palabras completas con mayúsculas en cada palabra no inicial. Prefijar con 'un'. 
-> Ejemplos: unCliente // unaLineaPedido
NOTA: En caso de un parámetro, poner solamente la inicial. Ejemplos: para cliente, c // para lineaPedido, lp

Interfaces
-> Palabras completas con mayúsculas en cada palabra no inicial. Prefijar con 'I'. 
-> Ejemplos: IOrdenable // ISerializable
